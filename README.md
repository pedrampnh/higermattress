# فروشگاه آنلاین هیگر (Higer Mattress)

فروشگاه کامل با ۲۷ مدل تشک، صفحه اختصاصی برای هر محصول، سبد خرید سمت سرور، پرداخت آنلاین از طریق زرین‌پال، و فرم تماس.

## اجرای محلی (روی سیستم خودتان)

نیازمندی: Node.js نسخه ۱۸ به بالا.

```bash
npm install
cp .env.example .env
npm start
```

سپس آدرس `http://localhost:3000` را باز کنید.

## قبل از آنلاین کردن سایت (مهم)

فایل `.env` را باز کنید و این مقادیر را با اطلاعات واقعی جایگزین کنید:

| متغیر | توضیح |
|---|---|
| `BASE_URL` | آدرس واقعی دامنه سایت شما، مثلاً `https://higermattress.com` |
| `SESSION_SECRET` | یک رشته تصادفی و طولانی برای امنیت نشست کاربران |
| `ZARINPAL_MERCHANT_ID` | مرچنت‌آیدی ۳۶ کاراکتری واقعی شما از پنل زرین‌پال |
| `ZARINPAL_SANDBOX` | تا وقتی مرچنت واقعی نگرفته‌اید `true` بگذارید؛ بعد از دریافت مرچنت واقعی و تست موفق، مقدارش را `false` کنید |
| `ADMIN_PASSWORD` | رمز عبور ورود به پنل مدیریت (`/admin`) — حتماً قبل از آنلاین شدن عوض کنید |
| `SUPPORT_PHONE` / `SUPPORT_WHATSAPP` | اطلاعات تماس واقعی که در صفحه تماس با ما نمایش داده می‌شود |

نکته مهم درباره پرداخت: تا وقتی سایت روی یک دامنه و هاست واقعی و آنلاین قرار نگیرد، زرین‌پال نمی‌تواند کاربر را به `callback_url` شما برگرداند. یعنی تست کامل پرداخت فقط بعد از آپلود روی هاست ممکن است.

## هاست کردن سایت

این یک اپلیکیشن Node.js معمولی است (نه یک سایت فقط HTML)، پس نیاز به هاستی دارید که از Node.js پشتیبانی کند. گزینه‌های ساده و ایرانی/بین‌المللی:

- **لیارا (Liara)** — ساده‌ترین گزینه برای دیپلوی سریع اپ‌های Node.js با پرداخت ریالی
- **Render** یا **Railway** — گزینه‌های بین‌المللی با پلن رایگان برای شروع
- هر VPS معمولی + PM2 برای اجرای دائمی `node server.js`

مراحل کلی: کد را روی هاست آپلود کنید → `npm install` → متغیرهای `.env` را در پنل هاست تنظیم کنید → `npm start` (یا از طریق PM2).

## پنل مدیریت (بدون نیاز به دست زدن به کد)

بعد از اجرای سایت (`npm start`)، به آدرس زیر برو:

```
http://localhost:3000/admin
```

رمز عبور پیش‌فرض `higer-admin` است (توی فایل `.env` با کلید `ADMIN_PASSWORD` تنظیم می‌شه — حتماً قبل از آنلاین کردن سایت عوضش کن).

از داخل پنل می‌تونی:

- **محصولات**: تشک جدید اضافه کنی، قیمت هر سایز رو ویرایش کنی، یا محصولی رو حذف کنی
- **متن سایت**: عنوان و توضیح صفحه‌ی اصلی، اطلاعات تماس (تلفن، واتساپ، آدرس)، متن فوتر، و توضیح هر رده‌ی کیفی (درجه ۱ تا ۴) رو ویرایش کنی
- **پیام‌های تماس**: پیام‌هایی که مشتری‌ها از فرم تماس با ما فرستادن رو ببینی
- **سفارش‌ها**: لیست همه‌ی سفارش‌ها، وضعیت پرداخت، و کد پیگیری تراکنش زرین‌پال رو ببینی

هر تغییری که توی پنل بدی، بلافاصله روی سایت اصلی (`/`) منعکس می‌شه، بدون نیاز به ری‌استارت کردن سرور.

## قیمت‌ها

قیمت محصولات در فایل `data/products.js` است. اعداد لیست قیمت اصلی به عنوان «هزار تومان» در نظر گرفته شده‌اند (مثلاً عدد ۲۵۰۰ = ۲,۵۰۰,۰۰۰ تومان). اگر این فرض اشتباه است یا قیمت‌ها به‌روزرسانی شده، همین فایل را ویرایش کنید — هر بار که بخواهید قیمت یا موجودی تغییر کند، فقط این فایل نیاز به ویرایش دارد.

## ساختار سفارش‌ها و پیام‌ها

فعلاً سفارش‌ها در `data/orders.json` و پیام‌های فرم تماس در `data/messages.json` ذخیره می‌شوند (فایل ساده JSON، بدون نیاز به دیتابیس جداگانه). برای حجم بالای سفارش یا وقتی به پنل مدیریت نیاز پیدا کردید، می‌توان به یک دیتابیس واقعی (مثل PostgreSQL) مهاجرت کرد.

## ساختار پوشه‌ها

```
server.js               نقطه شروع اپلیکیشن
data/products.json       کاتالوگ ۲۷ محصول و قیمت‌ها (از پنل مدیریت یا مستقیم ویرایش می‌شه)
data/content.json        متن سایت (خودکار از پنل مدیریت ساخته می‌شود)
data/orders.json         سفارش‌های ثبت‌شده (خودکار ساخته می‌شود)
data/messages.json       پیام‌های فرم تماس (خودکار ساخته می‌شود)
routes/shop.js            تمام مسیرهای فروشگاه (محصولات، سبد، پرداخت، تماس)
routes/admin.js           تمام مسیرهای پنل مدیریت
middleware/auth.js        محافظت از مسیرهای پنل مدیریت با رمز عبور
utils/products-store.js   خواندن/نوشتن کاتالوگ محصولات
utils/content-store.js    خواندن/نوشتن متن سایت
utils/zarinpal.js         اتصال به درگاه زرین‌پال
utils/store.js            ذخیره و بازیابی سفارش‌ها/پیام‌ها
views/                    قالب‌های صفحات فروشگاه (EJS)
views/admin/              قالب‌های صفحات پنل مدیریت
public/css/style.css      استایل سایت اصلی
public/css/admin.css      استایل پنل مدیریت
```
